The operator has a `portfolio.json` file produced by `--transform` (optionally enriched by `--analyze`) and needs to share consumption health insights with stakeholders — Executive, EA, CSM, MU Lead — without requiring them to install any software or have internet access. This story delivers the `--dashboard` tool: the operator runs `consumption-analyzer --dashboard <portfolio.json>` and receives a single self-contained HTML file written to `data/`. The file implements a 3-pane left-drives-right layout using Bootstrap 5 and Bootstrap Icons inlined in the HTML. The left pane shows all industries with their aggregated annual_contract_value, budget_contract_value, and consumed_contract_value. Clicking an industry populates the middle pane with customer cards, each showing account_insights (Step 4), aggregated contract values, and EA insights per L1 (enterprise_architecture_insights, Step 3) in a collapsible section. Clicking a customer populates the right pane with the full L3 product detail grouped by L1 then L2, showing solution_architecture_insights (Step 2) per L2 and the per-year monthly series table plus ai_insights (Step 1) per L3 product. All AI insight fields display a "Run --analyze to generate insights" message when their array is empty.

A successful completion means the operator can open the generated HTML file in any browser without internet access, click through the industry → customer → product hierarchy, see all five levels of AI insight (when --analyze has been run) or graceful empty states (when it has not), and share the single file with any stakeholder. The dashboard completes the full PDF → CSV → portfolio.json → AI enrichment → shareable HTML tool chain.
