The operator has a `portfolio.json` file produced by `--transform` and needs to share consumption health insights with four distinct stakeholder audiences — Executive/Regional Head, MU Lead, CSM, and EA — without requiring them to install any software or have internet access. This story delivers the `--dashboard` tool: the operator runs `consumption-analyzer --dashboard <portfolio.json>` and receives a single self-contained HTML file written to `data/`. The file opens in any browser with no internet connection required because pure CSS (no Bootstrap) and Chart.js are inlined. The dashboard has four role-based tabs, each capped at six content sections, and every at-risk product in any tab displays its recommended action inline without requiring navigation to a separate page.

A successful completion means the operator can open the generated HTML file, navigate to any tab, and find the data their audience needs: the Executive tab shows portfolio KPIs, top 5 at-risk products by dollar value, a solution-area attainment heatmap, and the AI narrative (read from the portfolio JSON if present); the MU Lead tab shows solution-area breakdown with run-rate and predictability scores; the CSM tab shows the product-level table filterable by solution area with risk color-coding, trend arrows, and recommended actions; the EA tab shows zero-utilization products, overperformers, and high-variance products. The operator can email or share this single HTML file with any stakeholder and it works immediately, completing the full PDF-to-dashboard tool chain.
