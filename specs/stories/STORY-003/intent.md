The operator has a cACV portfolio file — either the `portfolio.json` produced by `--transform` or the raw CSV directly — and wants a domain-aware AI narrative that explains consumption health in SAP-specific business terms. They run `consumption-analyzer --analyze <file>` and receive a plain-text analysis using `anthropic--claude-sonnet-latest`, which has sufficient reasoning capability to interpret attainment patterns, identify which solution areas are under-consuming against target, and frame findings in terms a CSM or MU Lead would act on. The prompt template at `/src/ai/analyze.md` is rewritten for the cACV domain: it provides model context about fiscal year structure, what attainment means, how to read at-risk classifications, and what constitutes an actionable finding. Today's date is injected into every prompt so the model knows which months have reported actuals versus which are future. The `AIClient` class in `src/lib/aiClient.js` handles the SDK call using `MODELS.sonnet` and a default token budget of 2048 via `AI_MAX_TOKENS`.

A successful completion means the operator receives a coherent cACV-domain narrative on stdout that identifies the key attainment story for the portfolio, names specific solution areas or products of concern, and suggests what the data implies for account engagement. Both input modes must produce valid output: when given a `portfolio.json`, the model receives pre-computed metrics including risk classifications and attainment percentages; when given a raw CSV, the model receives the tabular data directly. This output is also consumed by `--dashboard` when `--analyze` is run before `--dashboard`, embedding the narrative in the Executive tab's AI narrative paragraph.
