The operator receives SAP cACV portfolio reports as PDF files. These PDFs contain a specific two-row column header structure followed by data rows where monetary values are comma-formatted. This story delivers the `--pdf2csv` tool that reliably extracts this exact structure: the two header rows are detected and merged into a single normalized header; comma formatting is stripped and values are parsed to unquoted numerics; page noise is suppressed. The operator runs `consumption-analyzer --pdf2csv <filename.pdf>` and the output CSV is written to the same directory as the input. Column names in the output CSV must map cleanly to the canonical field names expected by --transform: the ACV-actuals column must be labelled to produce `annual_contract_value`, the budget column to produce `budget_contract_value`, and the consumed/actuals column to produce `consumed_contract_value` after fieldMapper processing — no manual cleanup required.

A successful completion means the operator can take a real SAP cACV PDF, run the command, and receive a CSV whose column names and numeric values are directly usable as input to `--transform` without manual cleanup. The extracted CSV preserves the YYYYMM month column structure, outputs all monetary values as plain numbers, and produces exactly the columns that --transform's fieldMapper expects. The operator gains a reliable, scriptable first step in the full PDF → CSV → portfolio.json → AI enrichment → dashboard tool chain.
