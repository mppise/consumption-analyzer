The operator receives SAP cACV portfolio reports as PDF files. These PDFs contain a specific two-row column header structure ("Measures / cACV Target / cACV Actuals" over "SOLUTION_AREA / SUB_SOLUTION_AREA / LOGICAL_PRODUCT / LOGICAL_PRODUCT_ID / MONTH / $ / $") followed by data rows where monetary values are comma-formatted (e.g. "22,865"). This story delivers a rewritten `--pdf2csv` tool that reliably extracts this exact structure: the two header rows are detected and merged into a single normalized 7-column header; comma formatting is stripped and values are parsed to unquoted numerics; page noise (page-number rows, metadata cells) is suppressed. The operator runs `consumption-analyzer --pdf2csv <filename.pdf>` and the output CSV is written to the same directory as the input, named `<input-basename>.csv`. The output path is emitted to stdout on success. When extraction fails or no tables are detected, the tool exits with a clear error on stderr and a non-zero exit code.

A successful completion means the operator can take a real SAP cACV PDF, run the command, and receive a CSV whose column names and numeric values are directly usable as input to `--transform` without manual cleanup. The extracted CSV has exactly 7 columns (solution_area, sub_solution_area, logical_product, logical_product_id, month, cacv_target, cacv_actual), preserves the YYYYMM month column structure, and outputs all monetary values as plain numbers. The operator gains a reliable, scriptable first step in the full tool chain from raw PDF to portfolio dashboard.
