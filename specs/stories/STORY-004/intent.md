The operator has a cACV CSV file (produced by `--pdf2csv` or placed directly in `data/`) and needs all portfolio metrics computed and structured before the dashboard or AI analysis can run. This story delivers `--transform`: the operator runs `consumption-analyzer --transform <file.csv>` and receives a `portfolio.json` written to `data/` containing every computed metric — attainment percentages, YTD totals, trend direction, year-end forecasts, forecast confidence, contract utilization, and risk classification per the nine defined business rules — organised as a customer > solution_area > sub_solution_area > products hierarchy. All computation is deterministic Node.js; no AI is involved. The risk engine in `src/lib/risk.js` is the single authoritative implementation of the nine rules.

A successful run means the operator can open `portfolio.json` and see every product nested at the correct sub-SA level with accurate YTD figures, risk levels, and AI placeholder fields (`insight`, `recommendation`, `ea_action`) set to null. The `customer.risk_items[]` array lists deduplicated at-risk products for triage. The `_composite_key` internal dedup field is stripped before the file is written. This JSON becomes the sole input to `--dashboard` and `--analyze`, completing the data layer so those stories can proceed without re-parsing the CSV.
