The operator has a cACV CSV file (produced by `--pdf2csv` or placed directly in `data/`) and needs all contract metrics computed and structured into the portfolio.json format before the dashboard or AI analysis can run. This story delivers `--transform`: the operator runs `consumption-analyzer --transform <file.csv>` and receives a `portfolio.json` written to `data/` containing the full L1 → L2 → L3 product hierarchy with all contract months computed. For each `contract_month` the tool produces: `annual_contract_value`, `budget_contract_value`, `consumed_contract_value`, and the three variance fields (`acv_gap`, `budget_gap`, `budget_attainment`). All computation is deterministic Node.js using integer-safe arithmetic; no AI is involved. All AI insight fields (`contract.ai_insights`, `solution_architecture_insights`, `enterprise_architecture_insights`, `account_insights`, and `industry_insights[].summary`) are initialised as empty arrays ready for --analyze to populate.

A successful run means the operator can open `portfolio.json` and find every L3 product nested at the correct L2 level with accurate per-month contract figures and computed variances, all industry_insights entries present with the correct industry name, and all AI array fields set to `[]`. The portfolio JSON becomes the sole input to `--dashboard` and `--analyze`, completing the data layer so those stories can proceed without re-parsing the CSV.
