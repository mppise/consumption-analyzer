The operator needs a working, installable CLI binary before any feature can be built. This story establishes the complete project skeleton: the npm package with ESM module configuration, the commander-based entry point at `src/cli.js`, dotenv loading, the config module at `src/config/index.js`, the `deploy.sh` install script, and the test scaffold using Node.js built-in test runner. No conversion features are included — this is purely the shell that all future tool modules plug into.

A successful completion means the operator can run `./deploy.sh` from the project root, have `consumption-analyzer` become available as a system command, run `consumption-analyzer --help` and see a usage message, and run the test suite with `npm test` against an empty but passing test file. The architecture's dispatch pattern, exit code contract, and config loading convention are all in place and verified — every subsequent story can be built directly on top of this scaffold without revisiting these foundations.
